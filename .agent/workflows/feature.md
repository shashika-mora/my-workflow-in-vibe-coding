# Feature Implementation Workflow

This workflow standardizes the process of adding a new feature.

1. **Research**: 
   - Analyze the current `src/` directory to understand the existing logic and UI patterns.
   - Look at `PATTERNS.md` and `.agent/rules/style.md` to ensure compliance.
2. **Plan**:
   - Propose design variations or architectural plans for the requested feature. Do not write code yet.
   - Detail which files will be modified, created, or deleted.
3. **Review**:
   - Wait for user selection and approval of the plan.
4. **Implement**:
   - Generate the code for the selected variation.
   - Adhere to the established styling framework and typing conventions.
5. **Verify**:
   - Run the dev server.
   - Capture screenshots or videos if UI features were added.
   - Run unit tests and provide results.
