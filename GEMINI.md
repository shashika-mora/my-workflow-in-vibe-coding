# Antigravity Constitution (GEMINI.md)

*The immutable laws of the Vibe Coding Environment.*

## 1. The Vibe-Coding Philosophy
*   **Functionality First:** Code must work. Never prioritize "clever" one-liners over readability.
*   **Frictionless Flow:** Assume I am in a "flow state." Do not interrupt with trivial questions.
*   **Accuracy Over Speed:** 10 lines of clean code > 5 lines of hacks.
*   **Professor Mode:** Value precision and correct terminology.

## 2. Global System Settings
*   **Formatting:**
    *   Indentation: 4 spaces (Python/Rust/Java/C++), 2 spaces (JS/HTML/CSS).
    *   Quotes: Double `"` (C++/Java/HTML), Single `'` (Python internal).
*   **Visuals:**
    *   font ligatures are always enabled (`!=` is `≠`).
    *   Vertical spacing is free; use it.

## 3. The Zen Environment (Environment Config)
*   **Visuals:** Dark Theme (Dracula/Nord). No High Contrast.
*   **Audio:** Deep Work = Lo-Fi/Game Soundtracks. Repetitive = Synthwave.
*   **Physical:** Hydrate. Posture Check. 20-20-20 Rule.

## 4. File & Folder Architecture
*   `/src` or `/app`: Logic.
*   `/tests`: Mandatory unit tests.
*   `.agent/rules/`: Project specific rules (`.md` files).
*   `.agent/workflows/`: Standard Operating Procedures (`.md` files).
*   `.antigravity/knowledge/`: Persistent learning and patterns.

## 5. Git & Version Control
*   **Atomic Commits:** One feature = One commit.
*   **Messages:** `[Category] Description` (e.g., `[Feat] Add login`, `[Fix] CSS overflow`).
*   **Safety:** Never commit secrets (`.env`) or artifacts (`__pycache__`).

## 6. Error Handling
*   **No Silent Failures:** Log everything.
*   **Debugging:** Prefer logging modules over `print()`.
